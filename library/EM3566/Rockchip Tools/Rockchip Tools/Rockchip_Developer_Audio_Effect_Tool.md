概述

Audio Effect Tool为图形化调参工具，用于为Audio Effect工程配置参数。
| 模块切换以及基本参数配置 |     |     |
| ------------ | --- | --- |
图1 基本参数配置以及模块切换区域
首先是基本参数设计以及模块切换部分，该部分的总体界面如上图所示。
在红框所示区域，用户可以对基本参数进行设置：
单
| 参数名 | 描述  |     |
| --- | --- | --- |
位
| NumChan    | 输入通道数，1对应单声道，2对应双声道 | -   |
| ---------- | ------------------- | --- |
| SampleRate | 采样率                 | Hz  |
| BitRate    | 比特率                 | bit |
选中表示左右声道都使用左声道的参数设置，否则每个通道的参数需单独
| Link |     | -   |
| ---- | --- | --- |
设置
注意，Channel不是基本参数，其作用是选择用户接下来要设置参数的通道。
在蓝框所示区域，用户可以通过复选框决定是否使用对应模块。
最后是绿框所示区域，该部分体现了Audio Effect各模块数据流方向，Audio Effect总共有Auto Gain、Bass
Enhancer、Exciter、EQ、MultiBand DRC(MBDRC)、AGC以及Maximizer七个模块。数据流按以下顺序：
输入->SPK->AutoGain->Bass Enhancer->Exciter->EQ->MBDRC->AGC->Maximizer->输出。
对于各个子模块，用户通过点击按钮，就可跳转到对应模块的参数配置界面。而使用红框中的Channel下
拉选项，就可以在同一模块的不同通道之间进行切换。而Auto Gain模块由于自身计算的特点，左右声道
都是用同一套参数配置进行计算。而PreGain的设置参数则是放到了SPK里面。
用户可以通过左上角的"Load"和"Save"按钮读取或保存参数配置，还可以用Push直接将参数文件推到RK
的硬件中，以及用Pull将硬件中的参数文件读取出来。

SPK
图2 SPK子模块
该模块用于显示喇叭频响特性，左上角部分显示当前处理声道，channel1对应左声道，而channel2对应右

AUTOGAIN

图3 AUTOGAIN子模块
Auto Gain基于EBU R128准则对输入的多通道信号的响度进行测量，并以此为依据对信号的响度进行归一
化操作。用户可以通过输入框对参数进行配置，也可以通过"Reset"按钮将参数重置为默认值。目前
AutoGain是基于所有通道一起计算的，因此用户虽然可以设置不同通道的参数，但是该模块只有第一通
道的参数可以生效。
| 参数名          | 描述            | 单位  |
| ------------ | ------------- | --- |
| target_i_dB  | 响度归一化目标值      | dB  |
| attack_time  | 触发时间，控制幅度上升时间 | ms  |
| release_time | 释放时间，控制幅度下降时间 | ms  |
| r128_wins    | 测量响度时使用的窗长    | ms  |
| r128_steps   | 测量响度时使用的步长    | ms  |
注意，preGain不是AutoGain模块的参数，其是否生效不受该模块开关的控制。

MBDRC
图6 MBDRC子模块
MultiBandDRC模块由Crossover模块（红框）和DRC模块（蓝框）两部分组成，其中Crossover模块负责将
输入信号分频成不同的子频带信号，而DRC模块则对子频带信号进行动态范围控制，最后子频带信号合
并在一起输出。
Crossover
利用高阶滤波器，将输入信号分为不同频率范围的子频带信号。
单
参数名 描述
位
Cross 分出的子频带数目，设为2时进行二分频，设为3则进行三分配，设为4则进
-
Band 行四分配
【注意】
Low Frequency的freq_start固定为0，High Frequency的freq_end固定为fs/2，对这两者进行设置并不会
起作用。
当Cross Band为2时，中间的两个的DRC模块，即Med Frequency 1 和Med Frequency 2的参数无效，
在实际工程中不会起作用。而当Cross Band为3时，Med Frequency 2的参数无效，在实际工程中不会
起作用。
DRC
对子频带信号进行动态范围控制。

| 参数名             | 描述                       | 单位  |
| --------------- | ------------------------ | --- |
| freq_start      | 分频器开始频点                  | Hz  |
| freq_end        | 分频器结束频点                  | Hz  |
| gain_dB         | 分频器输出增益                  | dB  |
| drc_enable      | DRC使能开关，设为0则只分频，不进行DRC操作 |     |
| compress_start  | 压缩段起始点                   | dB  |
| expand_end      | 扩张段结束点                   | dB  |
| noise_threshold | 噪声门阈值                    | dB  |
| max_gain        | 模块允许的最大增益                | dB  |
| max_peak        | 模块处理后信号的最大峰值             | dB  |
| attack_time     | 触发时间，控制幅度上升时间            | ms  |
| release_time    | 释放时间，控制幅度下降时间            | ms  |
| hold_time       | 保持时间，当幅度下降时保持幅度统计时间      | ms  |
【注意】
在配置参数时，应保证noise_threshold<expand_end，expand_end<compress_start，(compress_start +
max_gain) < max_peak，否则工具会显示参数错误。

MAXIMIZER
图7 MAXIMIZER子模块
MAXIMIZER由AGC和Maximizer两部分组成，其中AGC负责对前面经过MBDRC处理的信号进行处理，
避免出现截幅情况，然后再经由Maximizer将信号的强度最大化。
AGC
对经过MBDRC后信号的动态范围进行控制。
| 参数名             | 描述                  | 单位  |
| --------------- | ------------------- | --- |
| compress_start  | 压缩段起始点              | dB  |
| expand_end      | 扩张段结束点              | dB  |
| noise_threshold | 噪声门阈值               | dB  |
| max_gain        | 模块允许的最大增益           | dB  |
| max_peak        | 模块处理后信号的最大峰值        | dB  |
| attack_time     | 触发时间，控制幅度上升时间       | ms  |
| release_time    | 释放时间，控制幅度下降时间       | ms  |
| hold_time       | 保持时间，当幅度下降时保持幅度统计时间 | ms  |
Maximizer
Maximizer先将信号的整体强度压缩到指定值，再将其整体放大到指定强度。

| 参数名           | 描述                 | 单位  |
| ------------- | ------------------ | --- |
| max_threshold | 压缩阶段目标强度           | dB  |
| ceiling       | 放大阶段目标强度，即模块输出最大幅度 | dB  |
| release       | 释放时间，控制幅度下降时间      | ms  |
【注意】
配置参数需保证max_threshold < ceiling

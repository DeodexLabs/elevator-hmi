1. 概述

1.1 ⾳频算法特性

多⻨克⻛阵列算法，包括回声消除AEC、波束形成BF、语⾳降噪ANR、⾃动增益AGC等。主要包括以下
应⽤场景：

回声消除AEC⽤于消除⻨克采集到的扬声器播放的声⾳
波束形成BF⽤于增强语⾳，抑制相⼲噪声和环境噪声
语⾳降噪ANR⽤于消除⻨克采集到的环境噪⾳
⾃动增益AGC⽤于增强语⾳电平信号

2. ⾳频算法说明

多⻨克⻛声学处理主要完成回声消除和语⾳增强任务。

2.1 ⾳频算法模块说明

完整的算法包括的流程模块如图1所⽰，包括如下部分：

2.1.1 辅助模块

这些模块包括⾼通滤波（High pass-Filter）、预加重（pre-emphasis）、去加重（de-emphasis）等模块，
其中，⾼通滤波模块⽤于滤除低频电路噪⾳，预加重⽤于对语⾳⾼频能量提升，增强传输过程⾼频信噪

⽐，在输出端再通过去加重进⾏还原。

2.1.2 AEC模块

AEC模块包括MDF模块和Delay模块。

MDF模块属于线性回声消除算法，⽤于消除回声
Delay模块为延迟估计模块，⽤于估计回采信号与⻨克⻛信号之间的延迟

2.1.3 Beamforming模块

Beamforming（以下简称BF）模块包括FAST_AEC模块、WAKEUP模块、ANR模块、Dereverb模块、AES
模块、AGC模块、CNG模块、DTD模块、Howling模块、EQ模块以及DOA模块。

FAST_AEC模块：线性回声消除算法，⽤于抑制残留回声
WAKEUP模块：RK定制唤醒，主要完成特定唤醒词唤醒机器
ANR模块：降噪模块，⽤于抑制环境噪声、⻛噪、冲击噪声

Dereverb模块：去混响模块，⽤于消除环境混响
AES模块：残留回声消除算法，⽤于进⼀步抑制残留回声；
GSC模块：波束模块，输⼊为多⻨⾳频时默认开启，当额外开启Fix/GSC/CHN_SELECT时，启⽤固
定⽅向波束/波束后滤波/定制化扩展mic；
AGC模块：⾃动增益模块，⽤于控制语⾳电平信号
CNG模块：舒适度噪声模块，⽤于添加舒适度噪声
DTD模块：双讲检测模块，⽤于检测语⾳通话状态
Howling模块：去啸叫模块，⽤于去除⾃激产⽣的啸叫；
EQ模块：均衡器模块，⽤于调整频段增益；
DOA模块：声源定位模块，⽤于定位声源⽅位；
AINR模块：AI降噪模块，⽤于抑制除⼈声外的噪声；

2.2 ⾳频算法流程

多⻨克⻛阵列算法提供两种不同的语⾳通路。

2.2.1 唤醒流程

该流程通过唤醒进⾏机器应答以及定位，输出唤醒⼈所在位置的⾳频，具体流程如图1所⽰。

图 1 唤醒流程系统框图

1. 阵列采集阵列信息后，先通过AEC模块消除线性回声，BF内置FAST_AEC的进⾏进⼀步的回声抑

制；

2. 经过固定波束输出增强后的语⾳；
3. 接⼊Dereverb模块去除残留混响；
4. 通过Wakeup唤醒模块定位出声源位置；
5. 将定位的语⾳语⾳送⼊GSC后处理模块进⾏后滤波，（注：蓝⾊箭头云端识别流程，直接将定位语

⾳送到AGC输出）；

6. 再将处理后的语⾳送到ANR模块进⼀步进⾏降噪；
7. 最后送到AGC模块进⾏语⾳增益，送到远端进⾏通话。

2.2.2 通话流程

该流程不进⾏唤醒，除了Wakeup模块外其余模块均与唤醒流程相似，具体流程如图2所⽰。

图 2 通话流程系统框图

1. 阵列采集阵列信息后，先通过AEC模块消除线性回声；
2. 接⼊ANR/AINR、Dereverb、AES模块分别进⾏噪声、混响估计以及残留回声消除；
3. 经过波束输出增强后的语⾳，同时⽀持声源定位以及定向拾⾳；
4. 最后送到AGC、EQ模块进⾏语⾳增益，送到远端进⾏通话。

3. ⾳频算法接⼝调⽤说明

该库的调⽤过程如下：

1. 设置参数：

该步骤⽤于对⻨克⻛算法开放的⾳频参数进⾏设置，其中，AEC相关参数的设置需要根据产品⾳频腔
体、器件特点以及使⽤需求等进⾏调整，使⽤时需要根据每个产品的特点设置参数，初步集成时可使⽤
默认参数，然后根据效果调整参数。参数设置需要通过rkaudio_preprocess.h⽂件调整，将该⽂件路径名作
为初始化的输⼊参数。

2. 初始化：

该步骤⽤于通话开启后，逐帧处理声⾳信号前。初始化接⼝函数名为rkaudio_preprocess_init，具体调⽤
如下所⽰：

st_ptr = rkaudio_preprocess_init(mSampleRate, mBitPerSample, num_src_channel,

num_ref_channel, &param);

其中，mSampleRate采样率⽀持8kHz、16kHz、24kHz、32kHz、44.1kHz、48kHz，mBitPerSample⾳频位
深为16bit，num_src_channel为mic通道数，num_ref_channel为回采通道数，param为输⼊参数，该参数通
过rkaudio_preprocess.h配置将在后⾯进⾏详细说明。

初始化完成算法内部参数初始化和相关内存申请，若初始化成功则返回st_ptr结构体，如果初始化失败，
结构体将为NULL。

3. 帧处理：

该步骤即是实时处理通话语⾳信号，其单位为帧，当固定每帧16ms、采样率为16k采样时，每帧包含256
个样点。帧处理包括接⼝函数为rkaudio_preprocess_short，具体调⽤如下所⽰：

out_size = rkaudio_preprocess_short(st_ptr, (short*)in, out, in_size / 2,

&is_wakeup);

其中，st_ptr为初始化后⽣成的结构体，in为输⼊⾳频，单位为byte，out为输出⾳频，单位为short，
in_size为⼀帧读⼊的byte数，即 256 * num_channel * 2 。

4. 销毁释放：

该步骤是在通话结束后，⽤于释放算法内存变量，接⼝函数为rkaudio_preprocess_destory(st_ptr)和
rkaudio_param_deinit(&param)。
